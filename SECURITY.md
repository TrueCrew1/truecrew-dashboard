# Security Policy

## Reporting a vulnerability

If you find a security vulnerability in this repository, please report it privately
rather than opening a public issue or pull request.

- **Email:** contact@truecrewllc.com — include a description of the issue, steps to
  reproduce, and any relevant logs/screenshots. Please don't include real customer
  data in a report.
- Please give us a reasonable amount of time to investigate and address the issue
  before any public disclosure.
- Do not test for vulnerabilities against production data you don't own, and do not
  publicly disclose or exploit an active vulnerability before it's been addressed.

## Scope

This covers the application code, API routes (`api/`), and infrastructure
configuration (`supabase/`, `vercel.json`, `netlify.toml`) in this repository.
Vulnerabilities in third-party dependencies should generally be reported upstream as
well — feel free to flag them here too if they affect this project specifically.

## What to expect

We'll acknowledge reports as soon as we're able to, investigate, and follow up with
next steps. There's no bug bounty program at this time.
